# 073-OfficialDocumentation — Documentación Oficial

## Descripción del dominio

Este módulo centraliza enlaces y referencias a la documentación oficial de las principales tecnologías, frameworks, lenguajes y herramientas del ecosistema de desarrollo. La documentación oficial es la fuente de verdad para APIs, configuraciones, mejores prácticas y ejemplos de uso. Mantener una colección actualizada de enlaces oficiales es esencial para cualquier equipo de ingeniería.

## Conceptos clave

- **Official Docs**: Documentación mantenida por los creadores o mantenedores oficiales de la tecnología.
- **API Reference**: Documentación detallada de endpoints, parámetros, tipos de respuesta y ejemplos.
- **Guías / Tutorials**: Documentación paso a paso para casos de uso comunes.
- **Release Notes**: Notas oficiales de cambios por versión publicadas por el equipo mantenedor.
- **Migration Guides**: Guías para migrar entre versiones mayores de una tecnología.
- **RFCs / Proposals**: Documentos de diseño y propuestas de cambios (ej. TC39, IETF RFCs).
- **Specifications**: Especificaciones formales del lenguaje o protocolo (ej. ECMAScript spec, OpenAPI spec).
- **Changelog**: Historial oficial de cambios del proyecto.
- **SDK / API Docs**: Documentación de librerías cliente oficiales.
- **Status Pages**: Páginas de estado de servicios cloud y APIs.

## Hoja de ruta

1. **Principiante**: Aprender a navegar la documentación oficial de tu lenguaje principal. Usar la barra de búsqueda y los índices. Leer guías de inicio rápido.
2. **Intermedio**: Dominar la API reference de tus frameworks. Consultar release notes y migration guides. Usar documentación offline (devdocs, Zeal, Dash).
3. **Avanzado**: Leer y entender RFCs y especificaciones. Contribuir a la documentación oficial de proyectos open-source. Mantener documentación interna alineada con la oficial.
