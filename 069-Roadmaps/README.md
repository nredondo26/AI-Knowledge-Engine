# 069-Roadmaps: Hojas de Ruta Tecnológicas

## Descripción del dominio

Las hojas de ruta tecnológicas (roadmaps) son planes estratégicos que describen la evolución esperada de una tecnología, desde su estado actual hasta su madurez futura. Este módulo contiene roadmaps detallados por tecnología específica (LLMs, RAG, búsqueda semántica, agentes, embeddings), por área de conocimiento (procesamiento de lenguaje, visión, sistemas de recomendación) y por perfil profesional (ingeniero de IA, arquitecto de ML, investigador). Incluye análisis de tendencias futuras (AI agents, on-device AI, model compression, AI regulation) y planes de adopción para organizaciones que buscan integrar estas tecnologías.

## Conceptos clave

- **Roadmap por tecnología**: LLMs (GPT-4 → GPT-5, Llama 3 → Llama 4, Mixture of Experts, Sparse models), RAG (naive → advanced → agentic RAG), Búsqueda (full-text → semántica → híbrida → generativa), Agentes (single agent → multi-agent → agentic organizations)
- **Roadmap por área**: NLP (BERT → GPT → Claude → especialización), Visión (CNN → ViT → multimodal → world models), Recomendación (collaborative filtering → deep learning → foundation models)
- **Roadmap por perfil**: Ingeniero de IA (fundamentos → frameworks → producción → arquitectura), Arquitecto de soluciones (diseño → escalabilidad → gobernanza → estrategia), Investigador (reproducción → contribución → liderazgo → visión)
- **Tendencias futuras**: Model Context Protocol (MCP), Agentes autónomos, AI agents como servicio, Small Language Models (SLMs), Test-time compute scaling, Neuromorphic computing, AI regulation y compliance (EU AI Act, Executive Order), AGI timelines
- **Plan de adopción**: Assessment inicial → prueba de concepto (PoC) → MVP → producción → optimización → escalamiento — incluye hitos, métricas de éxito, riesgos y mitigaciones
- **Evolución tecnológica**: Ciclo de Gartner Hype Cycle para cada tecnología — madurez real vs percepción del mercado — tecnologías emergentes vs establecidas — innovación incremental vs disruptiva
- **Matriz de habilidades**: Habilidades técnicas vs habilidades complementarias — niveles de competencia por rol — certificaciones relevantes — proyectos prácticos recomendados

## Hoja de ruta

1. **Principiante**: Entender un roadmap por primera vez — roadmap.sh: elegir un rol (AI Engineer, Data Scientist) y seguir las guías — identificar tecnologías clave vs accesorias — priorizar fundamentos sobre modas — crear un plan de aprendizaje personal de 6-12 meses
2. **Intermedio**: Construir un roadmap tecnológico para un equipo — evaluar madurez actual en cada tecnología (matrix) — identificar brechas de conocimiento — priorizar inversión en tecnologías con mayor ROI — roadmaps por proyecto (tecnologías necesarias para un caso de uso específico)
3. **Avanzado**: Roadmaps organizacionales de adopción de IA — fases: assessment → foundation → scaling → embedded — métricas de progreso (número de modelos en producción, cobertura de casos de uso, ROI acumulado) — roadmaps de migración tecnológica (p.ej., búsqueda textual → búsqueda semántica híbrida)
4. **Experto**: Roadmaps estratégicos a 3-5 años para C-level — análisis de tecnologías emergentes (quantum ML, neuromorphic, AGI) — roadmaps de transformación organizacional completa — plan de gestión de riesgos regulatorios — roadmaps de inversión en I+D vs adquisición de talento
