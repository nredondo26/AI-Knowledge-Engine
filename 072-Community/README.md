# 072-Community — Comunidades y Recursos

## Descripción del dominio

El dominio de comunidades abarca los espacios digitales y físicos donde desarrolladores, ingenieros y entusiastas de la tecnología colaboran, comparten conocimiento, resuelven dudas y contribuyen a proyectos. Incluye plataformas como Stack Overflow, GitHub Discussions, Discord, Reddit, foros especializados, meetups locales y conferencias internacionales. Participar activamente en comunidades acelera el aprendizaje, permite networking y fomenta contribuciones open-source.

## Conceptos clave

- **Stack Overflow**: Plataforma Q&A técnica con sistema de reputación, votos y badges. La mayor base de conocimiento de programación.
- **GitHub Discussions**: Foro integrado en repositorios de GitHub para debates, preguntas y anuncios.
- **Discord / Slack**: Canales de chat en tiempo real con comunidades temáticas por tecnología o proyecto.
- **Reddit (subreddits)**: Comunidades como r/programming, r/javascript, r/devops con discusiones y enlaces.
- **Dev.to / Hashnode**: Plataformas de blogging técnico donde desarrolladores publican tutoriales y experiencias.
- **Meetups**: Encuentros presenciales o virtuales organizados por tecnología o región (ej. meetup.com).
- **Conferencias**: Eventos como KubeCon, React Conf, AWS re:Invent, PyCon, JSConf.
- **Open Source Contributions**: Contribuir a proyectos open-source como vía de aprendizaje y visibilidad profesional.
- **Code of Conduct**: Normas de comportamiento en comunidades para mantener espacios inclusivos y respetuosos.
- **Reputation / Karma**: Sistemas de puntuación que reflejan la calidad de las contribuciones.
- **Mentorship**: Programas formales e informales de mentoría técnica dentro de comunidades.

## Tecnologías principales

- **Stack Overflow / Stack Exchange**: Red de sitios Q&A técnicos.
- **GitHub Discussions**: Foros integrados en repositorios GitHub.
- **Discord**: Plataforma de chat con servidores por tecnología, proyecto o región.
- **Slack**: Canales de comunicación para equipos y comunidades.
- **Reddit**: Agregador de noticias y foros temáticos (subreddits).
- **Dev.to**: Plataforma de blogging técnico open-source.
- **LinkedIn Groups**: Grupos profesionales de discusión técnica.
- **Twitter/X**: Red social para seguimiento de expertos y novedades técnicas.
- **Twitch / YouTube**: Streaming en vivo de coding, conferencias y talleres.

## Hoja de ruta

1. **Principiante**: Crear perfiles en Stack Overflow y GitHub. Leer y aprender de preguntas/respuestas existentes. Seguir subreddits y canales de Discord relevantes. Asistir a meetups locales.
2. **Intermedio**: Responder preguntas en Stack Overflow. Participar en GitHub Discussions de proyectos que uses. Publicar artículos en Dev.to. Presentar en meetups.
3. **Avanzado**: Organizar meetups o conferencias. Convertirse en moderador de comunidad. Ser mantenedor de proyectos open-source. Contribuir como speaker en conferencias internacionales. Mentorizar a nuevos desarrolladores.
