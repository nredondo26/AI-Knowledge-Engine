# 049-InterviewQuestions: Preguntas de Entrevista

## Descripción del dominio

La preparación para entrevistas técnicas es una habilidad esencial para profesionales de tecnología. Este módulo compila y organiza preguntas de entrevista categorizadas por tipo (algoritmos, system design, behavioral, específicas de tecnología), nivel de dificultad y rol (junior, semi-senior, senior, staff/principal). Incluye no solo las preguntas, sino estrategias de respuesta, criterios de evaluación, patrones de solución y recursos de preparación. Cubre entrevistas para empresas de tecnología (FAANG, startups, consultoras), procesos de system design, coding interviews, whiteboarding challenges y entrevistas de liderazgo técnico.

## Conceptos clave

- **Coding interviews**: Problemas de algoritmos en pizarra/editor, restricción de tiempo, evaluación de eficiencia (Big O), comunicación en vivo
- **Data structures & algorithms**: Arrays, strings, hash tables, trees (binarios, tries), graphs, recursion, DP, backtracking, sorting y searching
- **System design**: Diseño de sistemas a alto nivel (URL shortener, Twitter, Netflix, WhatsApp, Uber), escalabilidad, consistencia, disponibilidad, particionamiento
- **Low-level design / OOP**: Diseño de clases, patrones de diseño, principios SOLID, modelado de objetos (Parking Lot, Vending Machine, Chess Game)
- **Behavioral questions**: Preguntas situacionales (STAR method — Situation, Task, Action, Result), trabajo en equipo, conflicto, fracaso, liderazgo
- **Preguntas específicas de tecnología**: Deep dive en experiencia con React/Angular, Node/Python/Go, AWS/GCP/Azure, Docker/K8s, SQL/NoSQL
- **LeetCode patterns**: Two pointers, sliding window, binary search, BFS/DFS, topological sort, union-find, heap, monotonic stack, trie
- **Sistema de evaluación**: Signal vs noise, rúbricas (coding, system design, behavioral), hiring committees, calibrations
- **Whiteboarding**: Comunicación visual, dibujo de diagramas, flujo de datos, componentes, trade-offs
- **Take-home assignments**: Proyectos pequeños, REST APIs, microservicios, frontend challenges, CRUD completo, reglas y expectativas
- **Entrevistas de IA/ML**: ML system design, conceptos (bias-variance, regularization, loss functions), implementación de modelos, feature engineering, MLOps

## Hoja de ruta

1. **Principiante**: Dominar arrays, strings, hash tables — LeetCode Easy/Medium — entender Big O básico — preparar respuestas STAR para behavioral — practicar en HackerRank
2. **Intermedio**: Trees, graphs, recursion, BFS/DFS — LeetCode Medium/algún Hard — System Design básico (URL shortener, chat) — patrones de solución — mock interviews con Pramp
3. **Avanzado**: Dynamic Programming avanzada, tries, union-find — System Design complejo (distribuido, consistencia, sharding) — LeetCode Hards — deep dive en tecnología específica — diseño de APIs
4. **Experto**: System Design para staff+ (global scale, multi-region, cost optimization) — entrevistas de liderazgo técnico — mentoring de entrevistas — panelista en hiring committees — contribuciones al proceso de entrevista
